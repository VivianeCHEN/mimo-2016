Écosystème des langages de programmation

 

Quel langage pour quel usage ?

Pourquoi ?

Le langage naturel est ambigu et dépend du contexte.

Les langages de programmation sont des langages formels, développés pour décrire des algorithmes et des structures de données.

Diversité des langages

  • Premiers langages dans les années 50 (Fortran, Lisp, Cobol).
  • Plus de 400 langages actuellement.
  • Quelques propriétés :
    • Puissance d'expression
    • Typage faible / fort / statique / dynamique
    • Gestion mémoire, gestion d'erreurs, réflexion, ...
  • Différents paradigmes qui évoluent selon les besoins et la technologie.

Langage compilé ou interpreté

 

Un langage de programmation est mis en œuvre par un traducteur automatique: compilateur ou interpréteur.

Langage compilé

Un compilateur transforme un code source en un code cible (langage machine ou code intermédiaire).

Plus rapide mais demande une phase supplémentaire de compilation.

Exemples: C, C++, Java, ...

Langage interpreté

Un interpréteur transforme un code source "à la volée".

Plus souple mais demande la présence de l'interpréteur sur la machine.

Exemples: Python, Javascript, HTML, ...

Langage impératifs

Le paradigme impératif ou procédural est basé sur l'idée d'une exécution étape par étape (recette de cuisine).

  • Ensemble de structures pour contrôler l'ordre des commandes qui décrivent les étapes.
  • Abstraction réalisée à l'aide de procédures.
  • Une procédure principale, qui est la première à être exécutée.

Exemples: C, Pascal, Cobol, ...

Langage orienté objet

Dans le paradigme orienté objet, chaque objet est une entité active, qui communique par échange de messages.

  • Les objets sont groupés en classes
  • La programmation consiste à décrire les classes.
  • Structure hiérarchique avec héritage.

Exemples: C++, Java, Python, ...

Langage fonctionnel

Le paradigme fonctionnel est basé sur l'idée d'évaluer une formule et d'utiliser le résultat pour autre chose.

  • Evaluation des expressions en faisant appel à des fonctions.
  • Exécution étape par étape impossible.

 

Exemples: ML, Lisp, ...

Langage logique

Le paradigme logique est basé sur l'idée de répondre à une question par des recherches sur un ensemble, en utilisant des axiomes, des demandes et des règles de déduction.

  • Exécution du programme par évaluation.
  • Cascade de recherches de données dans un ensemble, en faisant usage de règles de déduction.

Exemples: Prolog, ...

Langages spécialisés

  • VRML (description de scènes 3D).
  • VHDL (description matérielle).
  • R (Statistiques).
  • DSL (langages dédiés comme HTML, CSS, SQL, ...).
  • Meta-programmation (programme pour écrire des programmes).

Typologie d'un projet Web

Coté Serveur

  • Différents OS: Unix/Linux, Windows, ...
  • Différents protocoles: HTTP, HTTPS, FTP, ...
  • Serveurs Web: Apache, Nginx, IIS, ...
  • Base de données: XML, SGBD, NoSQL, ...
  • Données statiques.
  • Données dynamiques (via Python): Contenu à rendre ou API.
  • Autres: API, Surveillance, déploiement, montée en charge, backup, SSO, Cron, ...

Coté client

  • Périphériques variés: ordinateur, mobiles, tablettes, ...
  • Navigateur variés: Chrome, Safari, Firefox, IE, ...
  • Fonctionnalités variées: Web vs App
  • Bandes passantes variées.
  • Technologies : HTML, CSS, Javascript.
  • Autres: Json, Token, Offline, Ajax, compatibilité, ...

La réalisation

  • Beaucoup de technologies en jeux
  • Spécifications technique et fonctionnelle
  • Gestion des sources
  • Gestion des bugs
  • Travail en équipe: méthodes (agiles)
  • Autres: livraison, veille technologique, maintenance, ...

Présentation des projets

Projet #1 : MOOC Podium

Site permettant d'afficher un "classement" des meilleurs MOOC sur un sujet donné à l'aides des informations trouvées sur le net (Twitter, web, open data, ...).

Projet #2 : Jeu de dame

Page web permettant de jouer contre l'ordinateur avec une interface simple.

Parties à deux en asynchrone.

Les données des parties sont stockées en base de données.

Projet #3 : Card Counting

Page web qui prend en entrée une partie de blackjack et calcule le prochain coup à jouer selon la méthode Hi-Low.

La partie est entièrement stockée en DB.

Projet #4 : Web Extractor

Une page qui prend une URL ou un fichier HTML en entrée, qui analyse le contenu et qui affiche des statistiques (graphiques et tableaux) sur les utilisations des balises ou sur le texte présent dans la base.

Des améliorations en terme d'analyse sémantique peuvent être prévues.

Projet #5 : Crawler thématique

Faire un crawler qui recherche des mots clefs dans une page.

Puis crawling des pages associés qui contiennent x% des mêmes mots jusqu'à un certain niveau.

Présentation des résultats

Projet #6 : API Tester

Page web qui prend un fichier de test en entrée au format XML ou JSON. Ce fichier contient la procédure de test de l'API souhaitée.

Le serveur exécute les tests et affiche un rapport. On peut accéder à l'historique des rapports.

Projet #7 : CorpuTweet

Analyser le contenu d'un fil Twitter et constituer un corpus avec toutes les citations.

Produire une visualisation du contenu des citations (description des types de média, une timeline et une petite analyse réseaux des ressources utilisées en SVG).

Projet #8 : What Are You Talking About

Page Web analysant un texte ou un site et permettant de classifier le contenu pour en extraire le type de discussion.

L'idée étant d'utiliser un classifieur "simple" sur des données.

Par exemple en entrée un ensemble de discussion d'un forum et permettre de classifier pour connaitre les types de sujets de discussions.

Projet #9 : TweetMap

en base de données XML sont stockés les infos de géolocalisation sur des tweets selon un certain critère, et la page affiche une carte indiquant les points des tweets.

Cela permet d'avoir une vue géographique des sujets abordés sur Twitter.
